Oberon User Manual

The best DeviantArt chat bot this side of the Mississipi.

Oberon is a DeviantArt chat bot written by bigmanhaywood. It's purpose was to fill the void for bots written in .NET. And considering there are none (at least none that are current), that void is pretty well filled. Oberon is the first full-featured bot to be written completely in the Microsoft .NET Framework and designed to be run on Windows. This means you don't even need a web server to run Oberon, or IIS or Apache...all you need is a Windows computer!

Oberon is an all-purpose bot designed with flexibility and extension in mind. It has a plugin system which allows developers to add to or modify the bot in any way they see fit. Oberon is very simple to set up and comes with a wide array of plugins ready for use, right out of the box. Plugins that are difficult or just straight impossible to create using a web-server based bot.

Sounds pretty sweet, eh?

Well, before we dive in, let's get the necessaries out of the way...

Installation

First, make sure that Microsoft .NET Framework 3.5 is installed on your system. Then download the Oberon zip file to wherever you want the files to live. Extract all the files to a directory.

You're all set!

Files for Oberon

The bot has a few files and directories that need to be present in order to run correctly. If you are having problems with the bot not starting or behaving incorrectly, then make sure the files below are all present.

Main files:

  • Oberon.exe
  • Plugins/System/Core.dll
  • Config/Log.config

Supporting libraries:

  • DeviantArt.Chat.Library.dll
  • log4net.dll
  • Microsoft.Practices.Unity.dll
  • Microsoft.Practices.ObjectBuilder2.dll
  • Ionic.Zip.dll

Configuration

If this is the first time you've configured the bot, follow the instructions below. Otherwise, see the normal config instructions.

1st-time Bot Configuration:

  1. Double-click Oberon.exe
  2. Enter your bot username. This must be a real dA account because it will be used to login to the chat network. It can be any existing dA account, even your own.
  3. Enter your bot password. This must the real password for the bot account.
  4. Enter your bot trigger. A 'trigger' is a string the bot looks for during a chat so that it knows it needs to do something.
  5. Enter the bot owner. This is the dA account of the user responsible for running the bot. This person will have super-admin privileges to the bot.
  6. Enter auto-join channels. Add a chatroom name that you want to the bot to join. Whenever the bot starts up it will join these rooms. Note: these can be changed later.
  7. Dance! You're all done!

Normal Bot Configuraton:

There's nothing essential to the running of the bot you are required to change. It will run fine as-is. But if you want to tweak some settings here's how to do it. After running for the first time the bot will save some config files. In the /Config directory open Bot.config.

<?xml version="1.0" encoding="utf-8"?>
<botSettings>
  <general>
    <add key="username" value="botusername" />
    <add key="password" value="botpassword" />
    <add key="trigger" value="%%" />
    <add key="owner" value="muysername" />
    <add key="debug" value="true" />
  </general>
  <about><![CDATA[<b>%N% %V%%S%</b> (%R% release)<br/><sub><b>Author:</b> :dev%A%:; <b>Owner:</b> :dev%O%:; <b>Trigger:</b> %T%<br/><b>%D%</b></sub>]]></about>
  <autoJoins>
    <add channel="thefreechat" />
  </autoJoins>
  <cookieData>
    <add key="uniqueid" value="3cc137a69e23188725e4bed9229aca50" />
    <add key="visitcount" value="0" />
    <add key="visittime" value="0" />
    <add key="username" value="botusername" />
    <add key="authtoken" value="3d63872ee10f3b20bc989536a2e0b614" />
  </cookieData>
</botSettings>

Here you can modify all the settings you put in the first time: username, password, trigger, autojoin rooms, etc. You can also change the about string so that when a user asks about the bot, it will respond with that information.

Note: in the about string you can enter some placeholders for bot information. Those place holders are:

  • %N% - The name of the bot
  • %V% - The bot release version
  • %S% - The bot status
  • %R% - The bot release
  • %O% - The bot owner
  • %A% - The bot author
  • %T% - The bot trigger
  • %D% - Whether or not the bot is running in debug mode

Logging Configuration

Oberon uses the library log4net to handle all of it's internal logging. The configuration for log4net lives in the /Config/Log.config file. The only setting need to be tweaked in this config file is the debugging level. In that config file you may see some XML similar to this:

<level value="INFO" />

You can change this value to any of the following:

  • ALL
  • DEBUG
  • INFO
  • WARN
  • ERROR
  • FATAL
  • OFF

I suggest using the setting "INFO". If you are have some strange behavior from the bot, set it to "DEBUG" to get extensive logging data. See the log4net online manual for a detailed explanation of rhe different logging levels.

Command Basics

We've covered the installation and configuration, let's get down now to the nitty gritty. The way to manually interact with the bot via a chatroom is by using commands. A command is any string that a user writes in the chatroom that starts with the bot's trigger. The bot's trigger is configured when you first run the bot, and can be changed by changing the /Config/Bot.config file. It can be any string. So for example, let's say our bot's trigger wa "~". A command might be:

13:18:14 <bigmanhaywood> ~time

What we've done here is ask the bot what it's current time is. It saw the trigger at the start of the message and knew that we were talking to it. Now that we've "triggered" a response, in the chatroom, the bot will respond with it's current time.

Command Help

Keeping track of each command in the system and what it does can take a lot of work. And that's where command help comes in. Each command has help text associated with it that explains what it does and how to use it. To see help for a particular command type the bot trigger followed by the word 'help':

03:08:00 PM <bigmanhaywood> !help time
03:08:00 PM <OberonBot> bigmanhaywood: 
  • Summary: Displays the current bot time.
  • Usage:
    time

Just replace "time" with the command you're interested in. Another way to get command help text is to type:

03:08:00 PM <bigmanhaywood> !time ?
03:08:00 PM <OberonBot> bigmanhaywood: 
  • Summary: Displays the current bot time.
  • Usage:
    time

Basically, just the command name followed by a question mark.

Command Security

The problem with a user issuing a command in the chatroom is that any user can send the command. That is why all commands have an access or privilege level. The access levels range from 1 to 100. The bot comes with some pre-configured 'priv classes' which are assigned default access levels for you to use:

  • Owner - Level: 100
  • Operators - Level: 99
  • Moderators - Level: 75
  • Members - Level: 50
  • Guest - Level: 25
  • Banned - Level: 1

These of course can be changed as much or little as you want.

Whenever an unknown user sends a command to the bot they are recognized as a guest and only have access to commands that are at the guest level. The bot operator can register users with the bot, each with their own access level.

Note: The bot operator can change the 'Guest' priv class, but anonymous users are slways assigned an access level of 25.

Plugins

The bot code in Oberon.exe contains most of the core structure for the system. But Oberon was designed for flexibility and extension in mind. So all commands reside in a "plugin". A plugin is what it sounds like; a piece of software that extends the capabilities of a larger program. In Oberon, plugins are responsible for making the bot do things.

Each command resides inside of a plugin. The Oberon Core plugin is responsible for allowing the bot to do all the normal things a user would do inside a chatroom. The Oberon Commands plugin is responsible for allowing a bot operator to manage the bot's basic settings.

Any plugin (except core plugins) can be activated or deactivated. By default all plugins are deactivated. A bot operator can change the activated status of a plugin at any time. An important thing to note is that if a plugin is deactivated you cannot access it's commands.

Any plugin located in the /Plugins will be loaded every time the bot runs, even if it's not activated. This may slow down the bot a little. Place a plugin in the /Plugins/_Off directory if you do not want the plugin to be touched by the bot at all.

Now that we've touched on the basic of a plugin, let's talk about the major commands in each of the provided plugins.

dAmn Core Plugin

This plugin handles the core system events. There are no commands for this plugin.

Oberon Command Plugin

This class encapsulates all internal Oberon commands. Let's touch on the basic commands for the system.

help - Display help on various commands.

help [command] - Gives help on commands

Example usage: !help access
about - Display infomation about the bot.

about - Display Bot type and owner
about uptime - display the amount of time the bot has been running
about system - display information about the operating system

Example usage: !about uptime
access - Changes the access level prileges for a command.

access [command] (Range 0 through 100)
access [command] default

Example: !access ping 0
Example: !access ping 100 - Changes access level for the 'ping' command to level 100. Now only a user with access level privileges of 100 will be able to execute the ping command.
autojoin - Adds or removes a chatroom from the auto join list.

autojoin add [room] - Adds a room to the autojoin list.
autojoin remove [room] - Removes a room from the autojoin list.
autojoin list - Displays all rooms set to autojoin.

Example usage: !autojoin add botdom
commands - List of available commands and their access levels.

commands - list of commands you have access to
commands all - list all commands
commands details - list all commands and the plugin they belong to

Example usage: !commands
ctrig - Temporarily change the bot's trigger.

ctrig [trigger]

Example usage: !ctrig # - changes the bot's trigger from '!' to '#'
ignore - Manages users the bot is set to ignore.

ignore list - Lists ignored users
ignore add [user] - Adds a user to the ignore list
ignore remove [user] - Removes a user from the ignore list

Example usage: !ignore list
join - Makes the bot join a chatroom.

!join [room]
part - Makes the bot leave a chatroom.

!part [room]
kick - Kicks a user from a chatroom.

kick (#room) [user] (reason) - Kicks a user from the chatroom. '#room' and 'reason' are optional. Bot must have kick privileges.

Example usage: !kick botdom devartuser
list - List the users in a chatroom.

!list [room]
priv - Manage bot priv class access levels.

priv list - Lists all named bot priv classes and their access levels
priv [priv class] [level] - updates an existing priv class with a new access level

Example usage: !priv guest 1 - Sets the priv class 'guest' to access level 1. All commands or users who had this level are also updated.
Example usage: !priv list - List all named bot priv classes
quit - Shuts down the bot.

!quit
restart - Restarts down the bot.

!restart
sudo - Allows a command to be executed as if it was sent from another user. Use very carefully. This command could allow users access to commands they don't usually have access to.

sudo [user] [command] - Execute [command] as if it was from [user].

Example usage: sudo devartuser ping - executes the ping command as if 'devartuser' had sent the command.
user - Manage users registered to the bot.

!user add [user] [level] - Register a user with the bot. Level can be an int or bot privclass.
!user edit [user] [level] - Edits a user in the bot user list. Level can be an int or bot privclass.
!user list - List all registered users
!user del [user] - Remove a user from the bot user list
!user addprivclass [name] [level] - Adds a new user priv class
!user delprivclass [name] - Removes a user priv class

Example usage: !user add devartuser 50

This plugin gives the bot the capability to respond to users. You can manage the bot responses per-chatroom.

Extras

This plugin package contains the following plugins:

  • Banned Words - Plugin that allows operator to kick users who use banned words within the chatroom.
  • Calc - Class the performs simple math calculations.
  • Dictionary - Plugin that lets a user find the definition of a word.

  • Regex - Plugin that lets a user test a regex pattern.
  • Responses - This is a bot response module for Oberon.

Last.fm

This plugin that allows the user to have some Last.fm functionality in the chatroom. Namely, looking up songs, albums, and lyrices.

Messaging

This plugin package contains the following plugins:

  • Away Message - Plugin that allows users in a chatroom to use the bot to manage their away message.
  • Bot Away - Plugin that allows the bot to be used for away messages.
  • Idler - Plugin to allow room administrators to perform an action on users who idle for too long.
  • Welcome Message - Welcome plugin to allow welcome message for a chatroom.

Notify

This plugin package contains the following plugins:

  • Flash Notify - Plugin that will flash the taskbar button when the bot owner receives a message.
  • Voice Notify - Plugin that will speak the notification.
  • MSN Notify - This plugin is not complete. Disregard it.

Search

Plugin that allows a user to perform a web search using the Bing API.

FAQ - Frequently Asked Questions

All righty, so now onto some common questions!

  1. How do I add a user to the bot so my friends can use it?

    Answer: See the user command. You can add users and specify their access levels that way. You can also use the access command to make a command accessible for anybody.

  2. What is Updater.exe?

    Answer: Udpater.exe is a program which the bot uses to update itself. The bot will detect when a new version is released and run Updater. It cannot be started manually.
